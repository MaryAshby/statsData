A1
And then the great diety said, "Create a funtion to get the grade from a quiz object".

And because they were a kind diety, the answer was given as such: 

var getGrad=function(quiz) {return quiz.grade;}

The function could be tested with: 

getGrade(classroom[0].quizes[0])

Where the "0" references the first item in the data sets given
i.e. classroom[0] is the first penguin in class and quizes[0] is the first quiz of said penguin.

But what isthis function thing?
The Murray text, Interactive Data Visualization for the Web, paints a lovely image on page 85 that values want to be held in the "kind, caring function's" curly brackets. Without the function they are undefined, with no true path in this world.

A2
One quiz grade is not enough! You must find the mean of the quiz grades for the first penquin - who will henceforth be known as "Kevin".

Again, the diety assisted with the creation of this kind, caring function. A new tool was given. THE GREAT D3.

var data=classroom[0].quizes.map(getGrade) var finalAnswer=d3.mean(data)

In the first question, we used Kevin as an example to retrieve his quiz grade and saved it in a variable named "getGrade". Now Kevin's mean quiz grades must be calculated with two new tools - map and d3.mean. Kevin = classroom[0] and his quizzes are mapped to getGrade so that the mean can then be calculated and returned.

A3
The mean of quizzes for Kevin is not enough! You must now creat a function to find the mean quiz grade for any of the penguins!

In this manner we were taught to create the world of penquin. 

var getPenguin=function(penguin){var pq=penguin.quizes.map(getGrade); var answer=d3.mean(pq); return answer;}

And it can be tested with our faithful Kevin. 

 var answer=getPenguin(classroom[0])
